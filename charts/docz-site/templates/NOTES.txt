The container serves the single-page app and, on the same origin, reverse-
proxies /api, /auth, /webhooks and /openapi.yaml to docz-api at
{{ .Values.config.doczApiUrl }} — so the browser and API share one origin
and the httpOnly session cookie is first-party (no CORS).

     kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "docz-site.fullname" . }} 8080:{{ .Values.service.port }}

2. Then browse:

     Site:      http://localhost:8080/
     Liveness:  http://localhost:8080/healthz
